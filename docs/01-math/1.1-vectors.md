## 1. 벡터란?

벡터는 그냥 **숫자 몇 개의 묶음**입니다. 3D 벡터는 `(x, y, z)` 세 숫자죠.
그런데 같은 묶음이 두 가지 의미로 쓰입니다. 이걸 구분하는 게 첫 단추입니다.

- **위치(점, position):** 공간의 한 지점. 예: 큐브의 꼭짓점 `(0.5, 0.5, 0.5)`.
- **방향(direction):** 어디를 향하는 화살표. 예: 빛이 오는 방향 `(0.4, 0.8, 0.3)`.

같은 `Vector3` 타입이지만, **위치인지 방향인지**에 따라 다루는 법이 달라집니다 (0.2에서 본 `w=1` vs `w=0`을 기억하세요). 코드에서는 같은 타입이라 **우리가 의미를 알고 써야** 합니다.

## 2. 기본 연산 — 성분별 계산

대부분의 연산은 그냥 **같은 자리끼리** 계산합니다.

```
a + b = (ax+bx, ay+by, az+bz)      방향을 이어 붙이기 / 점을 옮기기
a - b = (ax-bx, ay-by, az-bz)      "b에서 a로 가는 방향" = a - b
a * s = (ax*s, ay*s, az*s)         길이를 s배로 (방향 유지)
```

특히 **`목표 - 시작 = 시작에서 목표로 향하는 방향`** 은 그래픽스에서 끊임없이 나옵니다.
예: "표면에서 빛으로 가는 방향" = `빛위치 - 표면위치`. (Part 4 조명에서 바로 씁니다.)

```cpp
Vector3 Vector3::operator+(const Vector3& v) const { return { x + v.x, y + v.y, z + v.z }; }
Vector3 Vector3::operator*(float s) const          { return { x * s, y * s, z * s }; }
```

## 3. 길이와 정규화

벡터의 **길이(크기)**는 피타고라스 정리입니다.

```
|v| = √(x² + y² + z²)
```

방향만 필요할 때(예: "빛이 어느 쪽?")는 길이를 1로 맞춥니다. 이걸 **정규화(normalize)**라 하고, 결과를 **단위 벡터(unit vector)**라 부릅니다.

```
normalize(v) = v / |v|        길이로 나누면 방향은 그대로, 길이만 1
```

```cpp
float Vector3::Length() const { return std::sqrt(Dot(*this)); }   // √(v·v) = |v|

Vector3 Vector3::Normalized() const {
    float len = Length();
    if (len <= 1e-8f) return { 0, 0, 0 };   // 0벡터는 방향이 없음 → 0으로
    return { x / len, y / len, z / len };
}
```

> **왜 0 검사를 하나?** 길이가 0인 벡터를 정규화하면 0으로 나누기(NaN/무한대)가 됩니다. 그래픽스 버그의 흔한 원인이라 습관처럼 막아둡니다.
> **`Length()`가 `Dot(*this)`를 쓰는 이유:** `v·v = x²+y²+z²` 이므로 `√(v·v) = |v|`. 내적의 첫 응용입니다.

## 4. ★ 내적 (Dot Product) — "얼마나 같은 방향인가"

이번 레슨에서 **가장 중요한 개념**입니다. 조명, 카메라, 컬링 등 어디에나 나옵니다.

```
a · b = ax*bx + ay*by + az*bz       (계산은 이렇게 간단)
```

하지만 **기하학적 의미**가 핵심입니다.

```
a · b = |a| |b| cosθ                (θ = 두 벡터 사이의 각)
```

두 벡터가 **단위 벡터**라면 `|a|=|b|=1`이므로:

```
a · b = cosθ
```

즉 내적은 **두 방향이 얼마나 나란한지**를 -1 ~ 1 사이 숫자로 알려줍니다.

| 두 단위 벡터의 관계 | θ | a · b |
|---|---|---|
| 같은 방향 | 0° | **1** |
| 수직 | 90° | **0** |
| 반대 방향 | 180° | **-1** |

```cpp
float Vector3::Dot(const Vector3& v) const { return x * v.x + y * v.y + z * v.z; }
```

> **미리 보는 응용 (Part 4 조명):** 표면 법선 `N`과 빛 방향 `L`이 둘 다 단위 벡터일 때,
> `N · L`은 "빛이 표면을 얼마나 정면으로 때리는가"입니다. 정면(1)이면 밝고, 빗겨가면(0) 어둡습니다.
> 이게 바로 가장 기본적인 조명 공식 **램버트(Lambert) 모델**입니다. cube.hlsl에서 `saturate(dot(N, L))`로 등장합니다.

## 5. ★ 외적 (Cross Product) — "둘 다에 수직인 방향"

내적은 결과가 **숫자**였지만, 외적은 결과가 **벡터**입니다.

```
a × b = (ay*bz - az*by,
         az*bx - ax*bz,
         ax*by - ay*bx)
```

기하학적 의미: `a × b`는 **a와 b 둘 다에 수직인** 새 벡터입니다.

```
        a × b  (위로 솟음)
          ↑
          │
          └───────→ b
         ╱
        a
```

**어디에 쓰나?**
- **면의 법선 구하기:** 삼각형의 두 변을 외적하면 그 면이 향하는 방향(법선)이 나옵니다.
- **좌표축 만들기:** 카메라의 "오른쪽" 방향 = "위" × "앞" (Part 1.4에서 뷰 행렬 만들 때 핵심).

왼손 좌표계에서 우리 축들은 이렇게 맞물립니다:

```
Right × Up = Forward       (1,0,0) × (0,1,0) = (0,0,1)
```

```cpp
Vector3 Vector3::Cross(const Vector3& v) const {
    return { y*v.z - z*v.y,  z*v.x - x*v.z,  x*v.y - y*v.x };
}
```

> **주의:** 외적은 **순서가 중요**합니다. `a × b = -(b × a)` — 순서를 바꾸면 방향이 정반대가 됩니다.
> 카메라 축을 만들 때 순서를 틀리면 화면이 좌우/상하로 뒤집히는 단골 버그가 됩니다.

## 6. 코드 둘러보기 & 실행

[`Vector3.h`](../../src/math/Vector3.h)에는 자주 쓰는 방향 상수도 정의해 뒀습니다.

```cpp
static Vector3 Right()   { return { 1, 0, 0 }; }   // +X
static Vector3 Up()      { return { 0, 1, 0 }; }   // +Y
static Vector3 Forward() { return { 0, 0, 1 }; }   // +Z (화면 안쪽)
```
